- এই কোডটি হলো তোমার অ্যাপের সেটিংস বা কনফিগারেশন ফাইল। ঠিক যেমন মোবাইলের সেটিংসে গিয়ে আমরা রিংটোন, ডিসপ্লে বা প্রাইভেসি কন্ট্রোল করি, ফ্লাস্ক (Flask) অ্যাপেও এই ফাইলটি দিয়ে ডেটাবেস, সিকিউরিটি এবং ফাইলের লিমিট কন্ট্রোল করা হয়।

- ১. লাইব্রেরি ইমপোর্ট এবং ফোল্ডার পাথ

Python
import os
from datetime import timedelta

basedir = os.path.abspath(os.path.dirname(__file__))
কাজ: * os মডিউলটি অপারেটিং সিস্টেমের (Windows/Linux) বিভিন্ন কাজ করতে সাহায্য করে।

timedelta দিয়ে আমরা সময়ের হিসাব (যেমন- কয়দিন বা কয় ঘণ্টা) করি।

basedir: এটি তোমার প্রজেক্ট ফোল্ডারের একদম মূল বা 'Root' ঠিকানাটা বের করে রাখে।

  __file__ হলো এই কারেন্ট ফাইলটা। অন্য পিসিতে বা সার্ভারে রান করলে ফোল্ডারের লোকেশন যেন ভুল না হয়, তাই os.path.abspath দিয়ে আমি ডায়নামিকভাবে প্রজেক্টের মেইন লোকেশনটা basedir এ সেভ করে নিয়েছি।"


- ২. মূল ব্লুপ্রিন্ট: Config ক্লাস

class Config:

এটি হলো বেস বা মূল ক্লাস। এর ভেতরে থাকা সব সেটিংস ডিফল্টভাবে পুরো অ্যাপে কাজ করবে।

২.১. সিকিউরিটি বা নিরাপত্তা

    SECRET_KEY = os.environ.get('SECRET_KEY') or 'section-tracker-super-secret-key-2024'

ফ্লাস্ক ইউজারদের কুকিজ (Cookies) এবং সেশনকে (Session) সুরক্ষিত বা এনক্রিপ্ট রাখতে এই 'চাবি' বা কি (key) ব্যবহার করে।

-  হ্যাকাররা যেন ইউজারের সেশন হ্যাক করতে না পারে, তাই এই সিক্রেট কি ব্যবহৃত হয়। প্রথমে এটি সার্ভারের এনভায়রনমেন্ট ভ্যারিয়েবল (os.environ.get) থেকে কি খোঁজার চেষ্টা করবে। যদি না পায়, তাহলে অ্যাপ ক্র্যাশ এড়াতে 'or' এর পরের ডিফল্ট কি-টি ব্যবহার করবে।"

২.২. ডেটাবেস কানেকশন

    SQLALCHEMY_DATABASE_URI = os.environ.get('DATABASE_URL') or \
        'sqlite:///' + os.path.join(os.path.dirname(basedir), 'section_tracker.db')
    SQLALCHEMY_TRACK_MODIFICATIONS = False

 অ্যাপের ডেটা কোথায় সেভ হবে তা বলে দেওয়া। এটি লোকাল ফোল্ডারে section_tracker.db নামে একটি SQLite ডেটাবেস তৈরি করবে।

- TRACK_MODIFICATIONS অপশনটি ফ্লাস্ক-এসকিউএলঅ্যালকেমির একটি ইভেন্ট ট্র্যাকার। এটি অন থাকলে অ্যাপ প্রচুর মেমরি (RAM) খায়, তাই পারফরম্যান্স ফাস্ট রাখতে আমি এটি False করে দিয়েছি।


২.৩. সেশন এবং ফাইল আপলোড

    PERMANENT_SESSION_LIFETIME = timedelta(days=7)
    
    UPLOAD_FOLDER = os.path.join(basedir, 'static', 'uploads')
    MAX_CONTENT_LENGTH = 16 * 1024 * 1024
    ALLOWED_EXTENSIONS = {'png', 'jpg', 'jpeg', 'pdf', 'docx', ...}

- * timedelta(days=7): একজন ইউজার লগইন করলে ৭ দিন পর্যন্ত লগড-ইন থাকবে।

- MAX_CONTENT_LENGTH: ১৬ মেগাবাইটের চেয়ে বড় ফাইল কেউ আপলোড করতে পারবে না।

- আপলোডের সাইজ লিমিট বাইটে (byte) দিতে হয়। তাই ১৬ মেগাবাইটকে ১০২৪ দিয়ে দুইবার গুণ করে (১৬ * ১০২৪ * ১০২৪) বাইটে কনভার্ট করেছি। আর ALLOWED_EXTENSIONS দিয়ে রেস্ট্রিক্ট করে দিয়েছি যাতে কেউ ক্ষতিকর .exe বা ভাইরাস ফাইল আপলোড করতে না পারে।"
